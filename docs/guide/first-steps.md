# 首次进入与基本使用

本页说明安装完成后的最小配置顺序。对应对象包括浏览器入口、`Profile`、电台连接、音频链路和操作员配置。

## 第一步：确认浏览器入口

无论部署形态是桌面版、Linux 服务器还是 Docker，首次检查都应包含以下项目：

- 浏览器是否能打开 TX-5DR 界面
- 管理员入口或令牌是否可用
- 主界面是否显示频谱、表格和操作员区域

该步骤用于确认 Web UI、服务端入口和认证链路已连通。

## 第二步：创建或选择 Profile

`Profile` 是主项目里电台配置与音频配置的基本单元。根目录 `CLAUDE.md` 说明，`ProfileManager` 负责 Profile 的增删改查和激活。

首次配置时，建议先建立一套最小可用的 Profile，至少包括：

- 电台连接方式
- 音频输入与输出设备
- 模式和频率相关设置

## 第三步：配置电台连接

当前项目文档列出的连接方式包括：

- Hamlib 网络连接
- Hamlib 串口连接
- ICOM WLAN
- 无电台模式

该步骤的检查目标是连接建立、状态读取和频率控制是否正常。

## 第四步：配置音频链路

建议按以下顺序验证音频：

1. 接收音频进入主机
2. 浏览器界面显示频谱或瀑布图
3. 解码结果出现在表格中
4. 发射音频输出设备完成配置

该顺序对应接收链路、显示链路、解码链路和发射链路的逐项确认。

## 第五步：创建操作员

TX-5DR 支持多操作员模型。即使当前只使用单个操作员，首次配置时也建议明确填写以下字段：

- 呼号
- 网格
- 频率偏移或相关参数
- 自动化策略

这些字段会直接影响操作员状态、发射文本和策略选择。

## 第六步：执行首次收发检查

建议按以下顺序执行：

1. 观察频谱和解码是否稳定
2. 检查电台控制是否正常
3. 检查操作员配置和发射周期
4. 再启用发射

该顺序用于把接收、控制和发射拆成独立检查项，便于定位问题位置。
