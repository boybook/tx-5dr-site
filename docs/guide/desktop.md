# 桌面版安装

桌面版发布物基于 Electron。`packages/electron-main` 负责桌面宿主进程，服务端与 Web 界面随应用一起分发，因此该形态适合在本地桌面环境中直接运行。

## 适用范围

- Windows、macOS、Linux 图形桌面
- 电台、音频设备与工作主机位于同一位置
- 首次验证电台连接、音频链路和浏览器界面

## 安装步骤

1. 打开桌面版发布页：<https://github.com/boybook/tx-5dr/releases/tag/nightly-app>
2. 下载适合平台的安装包
   - Windows：`.msi` 或 `.7z`
   - macOS：`.dmg`
   - Linux 图形环境：`.deb` / `.rpm`
3. 按操作系统的常规方式完成安装并启动应用
4. 启动后继续阅读 [首次进入与基本使用](./first-steps)

## 运行结构

桌面版启动后，本地主机会同时承载 Electron 宿主、服务端进程和 Web 界面。浏览器界面可以在应用窗口中使用；如果本地网络策略允许，也可以通过浏览器访问运行中的本地服务入口。

## 与服务器版的差异

桌面版的部署边界集中在单台桌面主机上，适合本地使用和初始配置。长期在线、独立域名、多人共用等场景通常需要切换到 [Linux 服务器安装](./linux-server) 所描述的部署方式。

## 桌面版的实时语音配置

桌面版现在也和 Docker、Linux 服务器版一样，统一从“系统设置 > 实时音频”配置 LiveKit 与 `ws-compat`。

### 常见选择

| 场景 | 推荐 |
| --- | --- |
| 只在当前桌面应用窗口里使用 | `ws-compat` 或 LiveKit + `局域网 / 本机` |
| 同一局域网里的浏览器访问这台桌面机 | 通常仍然用 LiveKit + `局域网 / 本机` |
| 通过复杂公网代理、FRP 或只转网页隧道访问桌面机 | 直接使用 `ws-compat` |
| 桌面机本身有真实公网可直连，且你明确知道网络拓扑 | 才考虑 `公网自动发现` 或 `公网手动指定` |

### 为什么桌面版通常优先用局域网模式

桌面版的典型场景是：

- 电台就在当前电脑旁边
- 浏览器和应用都在同一台机器或同一局域网
- 并不希望桌面机长期承担复杂公网入口

因此多数情况下，LiveKit 只需要使用“局域网 / 本机”即可；如果你根本不想处理额外媒体端口，直接切到 `ws-compat` 也完全可以正常工作。

### 修改后如何生效

桌面版保存实时音频设置后，会自动刷新托管运行配置；但要让内嵌 LiveKit 进程完整应用变更，通常直接重启整个应用最稳妥。

::: info 不建议手工修改内部 YAML
桌面版会在自己的配置目录里生成托管的 `livekit.resolved.yaml`。这个文件属于运行时产物，不建议手工编辑。
:::

### 什么时候不建议在桌面版上坚持使用 LiveKit

如果你的桌面版使用方式属于下面这些情况，通常应优先改用 `ws-compat`，或者考虑迁移到 Linux 服务器版：

- 需要通过 FRP 只转一个网页端口给公网访问
- 只能反代网站本身，无法额外开放 LiveKit 媒体端口
- 无法明确判断真正的公网出口 IP
- 希望长期稳定提供远程多人访问
