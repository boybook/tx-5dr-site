# 桌面版安装

桌面版发布物基于 Electron。`packages/electron-main` 负责桌面宿主进程，服务端与 Web 界面随应用一起分发，因此该形态适合在本地桌面环境中直接运行。

## 适用范围

- Windows、macOS、Linux 图形桌面
- 电台、音频设备与工作主机位于同一位置
- 首次验证电台连接、音频链路和浏览器界面

## 安装步骤

1. 打开桌面版发布页：<https://github.com/boybook/tx-5dr/releases/tag/nightly-app>
2. 下载适合平台的安装包
   - Windows：`.msi` 或 `.7z`
   - macOS：`.dmg`
   - Linux 图形环境：`.deb` / `.rpm`
3. 按操作系统的常规方式完成安装并启动应用
4. 启动后继续阅读 [首次进入与基本使用](./first-steps)

## 运行结构

桌面版启动后，本地主机会同时承载 Electron 宿主、服务端进程和 Web 界面。浏览器界面可以在应用窗口中使用；如果本地网络策略允许，也可以通过浏览器访问运行中的本地服务入口。

## 与服务器版的差异

桌面版的部署边界集中在单台桌面主机上，适合本地使用和初始配置。长期在线、独立域名、多人共用等场景通常需要切换到 [Linux 服务器安装](./linux-server) 所描述的部署方式。

## 桌面版的实时语音配置

桌面版使用和服务器版一致的内置实时语音链路：默认 `rtc-data-audio` WebRTC DataChannel，失败时自动回退到 `ws-compat`。不再启动或配置内嵌 LiveKit。

### 常见选择

| 场景 | 推荐 |
| --- | --- |
| 只在当前桌面应用窗口里使用 | 保持 `Auto`，无需额外网络配置 |
| 同一局域网里的浏览器访问这台桌面机 | 保持 `Auto`，确认系统防火墙允许 UDP `50110` |
| 通过 FRP 或只转网页的隧道访问桌面机 | 使用 `ws-compat`，或额外映射一个 UDP 端口 |
| 桌面机有固定公网入口且明确知道网络拓扑 | 映射公网 UDP，并在实时音频设置中填写外部 UDP 地址 |

### 为什么桌面版通常不需要额外配置

桌面版的典型场景是：

- 电台就在当前电脑旁边
- 浏览器和应用都在同一台机器或同一局域网
- 并不希望桌面机长期承担复杂公网入口

因此多数情况下保持 `Auto` 即可。只有需要让其他网络里的浏览器访问语音功能时，才需要考虑 UDP 端口、防火墙或 FRP 映射。

### 修改后如何生效

“系统设置 > 实时音频”中的传输策略和外部 UDP 地址会影响新建或重连的实时语音会话。为了让桌面宿主和内置后端都重新读取环境变量或端口设置，修改底层端口后建议重启整个应用。

### 什么时候使用 `ws-compat`

如果你的桌面版使用方式属于下面这些情况，通常应优先改用 `ws-compat`，或者考虑迁移到 Linux 服务器版：

- 需要通过 FRP 只转一个网页端口给公网访问
- 只能反代网站本身，无法额外开放 UDP
- 无法明确判断真正的公网出口 IP
- 希望长期稳定提供远程多人访问
